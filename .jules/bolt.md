## 2026-05-17 - ThreadPoolExecutor Starvation from time.sleep()
 **Learning:** In an async codebase using ThreadPoolExecutors via `asyncio.to_thread()`, using synchronous `time.sleep()` for frequent/long intervals inside the worker thread causes thread starvation. Selenium functions are inherently synchronous, meaning a wrapper function runs entirely in a thread. If the wrapper relies on many `time.sleep()`s, it ties up a thread for the entire lifespan of the process (e.g. scrolling the page and waiting for 500ms repeatedly).
 **Action:** Instead of making the entire wrapper synchronous and passing it to `asyncio.to_thread`, convert the wrapper to an `async def`. Wrap *only* the synchronous I/O operations (like Selenium `driver.get()` or `driver.execute_script()`) in `asyncio.to_thread()` and use `await asyncio.sleep()` for any waiting. This releases the executor thread back to the pool during sleep periods, significantly improving concurrent performance and avoiding thread exhaustion. Use `await asyncio.to_thread(getattr, obj, 'property')` for synchronous properties that trigger I/O.
